SubForge — Development Roadmap
Last updated: v0.17.0 (complete) — next: v1.0.0
================================================================================


v0.17.0 — Subtitle Format Conversion & Expanded Format Support
--------------------------------------------------------------------------------
  GOAL: Let users convert subtitle files between formats and add support for
  subtitle formats beyond SRT/ASS/VTT across all tabs. pysubs2 handles the
  heavy lifting — this is primarily UI and format coverage work.

  KNOWN ISSUES (carried from v0.16.0) — RESOLVED in v0.17.0
  - [FIXED] Transcribe tab: Timestamp column was too narrow — end time cut off.
    Column is now wider (280px default) and user-resizable (Interactive mode).
  - [FIXED] Transcribe tab: timestamp cells were not editable inline. Now
    editable; bad input is rejected and the original value is restored.

  SUPPORTED FORMATS (finalized after pysubs2 audit)
  Six text-based formats are supported everywhere in SubForge — Single File,
  Batch, and the new Convert tab. All handled via pysubs2; no custom parsers.

    SRT   (.srt)        — SubRip. Universal standard. Full fidelity.
    ASS   (.ass)        — Advanced SubStation Alpha. Rich styling. Full fidelity.
    SSA   (.ssa)        — SubStation Alpha (older ASS variant). Full fidelity.
    VTT   (.vtt)        — WebVTT. Web standard. Timing fine; no style round-trip.
    TTML  (.ttml)       — W3C Timed Text. Basic parser; complex styling is lossy.
    SAMI  (.sami/.smi)  — Legacy Windows Media Player format. Rudimentary parser;
                          flag as potentially lossy on input.

  Excluded: MicroDVD (.sub) — frame-based, requires video framerate to convert
  correctly. No framerate context available in a Convert tab. See MAYBE SOMEDAY.

  LOSSY CONVERSION WARNINGS (surface in UI, never silently degrade)
  - ASS/SSA → any other format: styling is not preserved. Warn the user.
  - TTML or SAMI as input: complex styling may not survive the round-trip.

  CONVERT TAB (dedicated tab, fully featured)
  - Single-file mode: drag-and-drop or Browse for one subtitle file. Input
    format auto-detected from extension. Output format chosen from a dropdown
    (all 6 formats). Output written next to source using same stem + new
    extension. Lossy warnings shown inline when the path is known to be lossy.
  - Batch mode: point at a folder, choose a target format, convert all
    supported subtitle files found. Output written next to each source file.
    Summary on completion: N converted, N skipped, N failed.

  EXPANDED FORMAT SUPPORT IN SINGLE FILE & BATCH
  - Single File tab file picker and drop target must accept all 6 formats.
    Detection, cleaning, and Save As must all work for the new formats.
  - Batch tab folder scan must pick up all 6 extensions. Format-specific
    parse failures flagged per-file, not silently skipped.

  - Add new strings to all 14 language packs.


v1.0.0 — Accessibility, Themes & Release
--------------------------------------------------------------------------------
  GOAL: Full accessibility pass, visual theme support, and the final polish
  that makes SubForge worthy of a 1.0 label. No new major features after this
  point — only bug fixes, stability, and the release candidate period.

  - Light theme and high-contrast theme toggle in Settings > General.
    Dark theme is the existing default. The toggle must reapply immediately
    without restart (same pattern as sensitivity slider). Store selection in
    settings.json.

  - Font size setting in Settings > General: Small / Medium / Large.
    Scales the UI text globally via QApplication.setFont or a stylesheet
    substitution. Must not break fixed-size widgets or icons.

  - Keyboard navigation audit. Every interactive element reachable and
    operable without a mouse. Tab order must be logical within each tab.
    All buttons, sliders, and dropdowns must respond to keyboard input
    correctly.

  - Screen reader compatibility. Call setAccessibleName() and
    setAccessibleDescription() on all major widgets. Test with Windows
    Narrator at a minimum. Provide meaningful labels for icon-only buttons
    and progress indicators.

  - Final branding pass. Every user-facing string, label, button text,
    and tooltip reviewed for consistency and clarity. Icon creation and added to the installer.

  - Translated READMEs. Generate README.es.md, README.fr.md, etc. (one per
    language pack) in a docs/ folder, linked from the top of the main README.
    AI-generated translations are fine. These are maintenance-light once
    written — only update when major features ship.

  - "Open data folder" button in Settings > About (or General). Opens the
    directory that holds settings.json, Whisper model cache, and logs in the
    system file manager. Platform logic:
      Windows frozen (.exe): %APPDATA%\SubForge
      Windows/any from source: repo root (where settings.json lives)
      macOS: placeholder — ~/Library/Application Support/SubForge (TBD when
             macOS packaging is tackled post-1.0)
      Linux: placeholder — ~/.local/share/SubForge (TBD with Linux binary work)
    Use QDesktopServices.openUrl(QUrl.fromLocalFile(path)) — cross-platform,
    no subprocess needed. The frozen/source distinction is already handled by
    the paths module; just expose it as a button.

  - Scheduled scanning. Let users configure a folder watch on a schedule
    (e.g. nightly, hourly). New subtitle files that appear since the last run
    are automatically cleaned using the cleaning settings the user chose at
    scheduling time. Results logged; user notified of what was changed.

  - Release candidate period. Tag an RC build, run a clean-machine install
    test, and give at least a few days for issue reports before tagging 1.0.0.


================================================================================
POST-1.0 — FUTURE FEATURES
================================================================================

  MACOS AND LINUX BINARIES
  - Source code is cross-platform Python, but binaries require per-platform
    packaging and testing.
  - macOS requires physical Apple hardware for code signing and testing.
    Not currently feasible — no Apple devices available.
  - Linux can be tested in a VM. Candidate for an early post-1.0 minor release.
  - GitHub Actions already has macOS/Linux jobs written — just needs enabling
    and a test pass when the time comes.
  - Any platform-specific code must be additive (wrapped in platform checks),
    never replacing the Windows-specific frozen .exe path.

  AUTO-UPDATE
  - Download and apply updates in-app rather than opening the browser.
  - Security implications (signature verification, rollback) need careful
    design before implementation.

  PLUGIN SYSTEM
  - User-written detection logic in Python, for edge cases the regex engine
    cannot handle.
  - Sandboxing and a defined API surface need design work before coding.

================================================================================
DECISIONS & STANDING RULES
================================================================================
  - Version format: v0.x.0 through 1.0.0, then semantic (1.0.1, 1.1.0, etc.)
  - Extract & Save filename format: [video stem].[lang].[ext]
    e.g. Movie.eng.srt — SDH tracks: Movie.eng.sdh.srt
  - SubForge is an accessibility tool. SDH removal and any feature that
    degrades the HoH experience must carry a visible warning and must never
    be enabled by default. Always default to the HoH-friendly option.
  - Track title normalization during remux is always on — no option.
    Encoder credits in track titles serve no user and actively hurt HoH
    users navigating subtitle menus.
  - Cleaning options are global (settings.json), not per-tab.
  - New UI strings must be added to all 14 language packs — no exceptions.
  - Module ownership is strict: ffmpeg/ffprobe path resolution in
    core/ffprobe.py; mkvmerge in core/mkvtoolnix.py; Tesseract in core/ocr.py.
    Path resolution logic must not bleed across module boundaries.
  - Check for Updates stays in the status bar permanently.
  - Settings is a toolbar dialog, not a tab.
  - No emojis on buttons except ⚙ on the Settings button.
  - os.execv fails in frozen (PyInstaller) apps — use QProcess.startDetached
    or relaunch via subprocess for anything that requires a restart.
  - Settings must write to AppData (installer) or the repo root (source),
    never to Program Files.
  - Subprocess console windows must be suppressed (creationflags on Windows).
