SubForge — Development Roadmap
Last updated: v0.11.0 (complete) — next: v0.12.0
================================================================================

COMPLETED
--------------------------------------------------------------------------------
v0.11.0 — Whisper Audio Transcription ✓ COMPLETE
  - Transcribe tab: dedicated tab for generating subtitles from video audio
    using faster-whisper AI, running fully offline with no API keys required
  - Model selection: tiny / base / small / medium / large with plain-language
    speed vs. accuracy descriptions; downloaded models marked with ✓
  - Language selection: auto-detect or manual ISO language code
  - SDH mode: non-speech audio annotations (music, applause, etc.) included
    when enabled; suppress_tokens and vad_filter tuned per mode
  - Save as .srt: write transcribed output as [stem].[lang].srt next to video
  - Remux into video: add transcribed track to MKV (mkvmerge) or MP4 (ffmpeg)
    alongside existing tracks
  - Settings > Paths: Whisper model directory entry (browse for folder;
    blank = default location in SubForge data dir)
  - CUDA GPU acceleration when torch + CUDA available; falls back to CPU int8
  - Subprocess isolation: transcription runs in a separate Python process to
    prevent CTranslate2 native libraries from conflicting with Qt's runtime
  - Flat model storage: models stored without HF Hub symlinks for compatibility
    with Windows machines that don't have Developer Mode enabled
  - 14 language packs fully updated for all new Transcribe tab strings
  - core/whisper.py: path/model management, transcription pipeline, SDH
    segment detection, SRT assembly — all independent of GUI

PLANNED
--------------------------------------------------------------------------------

v0.12.0 — Transcribe Tab: Inline Editing
  - Step 1: After transcription completes, show each subtitle line in the
    Transcribe tab in the same block-list format as the Single File tab —
    index, timestamp, and text visible per line.
  - Step 2: User can click any line to edit the text inline within the tab.
    Edits are held in memory and reflected in the subtitle data structure.
  - Step 3: Save as .srt and Remux into video both use the edited output,
    not the raw Whisper output. This allows correcting transcription errors
    before saving or remuxing.

v0.13.0 — Diagnostics & Polish
  - Error log viewer: button in Settings (About tab or dedicated Diagnostics
    tab) that opens the crash log if it exists, or shows "No errors logged
    yet" if it doesn't. Log path: %APPDATA%\SubForge\subforge_crash.log
    when running from the installer, or <repo_root>/subforge_crash.log
    when running from source.
  - Unified error logging: roll errors from all tabs (Single File, Batch,
    Video Scan, Image Subs, Transcribe) into the same log file, not just
    unhandled launch exceptions. Each entry should include a timestamp,
    the tab/module of origin, and the full traceback.
  - Log storage consistency: source-mode writes log to repo root (already
    writable, no AppData dependency); installer mode writes to
    %APPDATA%\SubForge\ (already correct for Whisper crash log).

v1.0.0 — Accessibility, Polish & Release
  - Uninstaller cleans up %APPDATA%\SubForge\ — this directory now holds
    Whisper models (potentially several GB) and the crash log. The uninstaller
    should prompt the user and offer to delete it, rather than leaving it
    silently behind. Implement via [UninstallDelete] in the .iss file with
    a user prompt, or a [Code] section for conditional deletion.
  - Light theme and high contrast theme (toggle in Settings)
  - Font size setting (small/medium/large)
  - Keyboard navigation audit — every action reachable without mouse
  - Screen reader compatibility — setAccessibleName/setAccessibleDescription
    on all major widgets (Windows Narrator, macOS VoiceOver)
  - Tooltip audit — every button has one
  - Windows installer appears behind other windows on launch — Inno Setup
    foreground focus issue. SetupMutex and WindowVisible both tried without
    success. May require a post-install launcher workaround.
  - Tesseract reads music note symbols (♪) as "J" — Tesseract has no glyph
    for ♪ in its training data and substitutes the nearest Latin character.
    Options: post-process OCR output replacing isolated "J" near known music
    context with ♪, or use a custom Tesseract training set. Low priority.
  - Image Subs sensitivity slider does not update track list colors — when
    slider is moved, the Track Detail pane re-renders correctly but the tree
    item color (red/orange/green) does not update to reflect the new threshold.
    Should call _update_tree_item_color() for all scanned tracks on slider change.
  - Language change dialog in Settings shows Yes/No buttons that are not
    translated — the dialog body correctly switches to the newly selected
    language but the QMessageBox standard buttons remain in the OS language.
    Fix by replacing QMessageBox.question with a custom dialog using
    STRINGS-based button labels in the newly selected language.
  - Additional translations — French, Dutch (detection profiles already exist)
  - GitHub Pages documentation site from README with tab screenshots
  - File association on macOS
  - Full test pass: Windows, macOS, Linux
  - Release candidate period before tagging 1.0.0
  - Final branding pass — every string, label, tooltip reviewed
  - Stop Scan button on Batch and Video Scan — graceful cancellation of an
    in-progress scan that preserves results collected so far and does not
    corrupt the report. The current Clear button is not reliable for this.
    Requires a cancellation flag on the worker thread.
  - Bottom status bar currently only reflects Single File tab activity — Batch,
    Video Scan, and Image Subs all manage their own internal status labels and
    do not update the app-level bar. Wire tab-switching to sync the bottom bar
    with the active tab's last status, or redesign to use a single shared bar.
  - Translated READMEs — once the feature set is stable, generate per-language
    README files (README.es.md, README.fr.md, etc.) in a docs/ folder and link
    them from the top of the main README. AI-generated to match the 14 language
    packs. Maintain once, rarely update after 1.0.

DECISIONS & STANDING RULES
--------------------------------------------------------------------------------
  - Check for Updates stays in status bar. Opt-in and visible. Non-negotiable.
  - Settings is a dialog opened from toolbar button, NOT a tab.
  - Settings button lives on far right of toolbar (⚙ Settings).
  - No emojis on buttons except the gear icon on Settings.
  - Sensitivity slider: no recommendations made — user-judgment based. Default configurable in Settings.
  - Version format: v0.x.0 through 1.0.0, then semantic (1.0.1, 1.1.0 etc.)
  - Remux supports MKV (mkvmerge) and MP4/M4V (ffmpeg). No mkvmerge for MP4.
  - Extract & Save filename: [video stem].[lang].[ext] e.g. Movie.eng.srt
    SDH tracks: Movie.eng.sdh.srt
  - File queue in Single File shows color only, no stale count badges.
  - SubForge is an accessibility tool — SDH removal and any feature that
    degrades the HoH experience must carry a visible warning and must never
    be enabled by default. Always default to the HoH-friendly option.
  - Track title normalization during remux is always on — no option.
    Encoder credits in track titles serve no user and actively hurt
    HoH users navigating subtitle menus.
  - Cleaning options are global (settings.json), not per-tab.
  - mkvmerge path lives in Settings > Paths, not in Video Scan tab.
