POST-1.0 — FUTURE FEATURES
--------------------------------------------------------------------------------
These are real planned features, not wishlist items. Priority order TBD.

  AI-ASSISTED SUBTITLE GENERATION (v1.x or v2.0)
  - Whisper AI integration for local, offline subtitle generation
      - Whisper runs entirely on-device — no cloud, no API keys,
        consistent with SubForge's fully local philosophy
      - User points at a video file, Whisper transcribes audio to
        subtitle blocks, SubForge's cleaning engine runs on output
      - SDH mode: generate with sound descriptions and speaker labels
        for deaf and hard of hearing viewers — the accessibility angle
        that no other desktop subtitle tool currently does well
      - Export directly as .srt/.ass or remux back into the video
      - This would make SubForge the only tool that both generates
        AND cleans subtitles in one local application
  - Model selection (Whisper tiny/base/small/medium/large) with speed
    vs accuracy tradeoff explained in plain language
  - Language detection or manual language selection for transcription

  SUBTITLE FORMAT CONVERSION
  - SRT ↔ ASS ↔ VTT conversion within the app
  - Batch format conversion across a library folder

  ADDITIONAL TRANSLATIONS
  - Community contribution system via GitHub for additional languages beyond French/Dutch

  OCR FOR IMAGE-BASED SUBTITLES
  - PGS/VOBSUB scan support via Tesseract
  - Complex dependency but would close the only remaining gap in
    Video Scan (image tracks currently listed but unscannable)

  ANDROID
  - BeeWare/Briefcase port
  - Genuinely complex, not a near-term priority

  AUTO-UPDATE
  - Download and apply updates in-app rather than linking to releases
  - Security implications need careful review before implementing

  PLUGIN SYSTEM
  - User-written detection logic in Python beyond regex
  - For edge cases the regex engine cannot handle


DECISIONS & STANDING RULES
--------------------------------------------------------------------------------
  - Check for Updates stays in status bar. Opt-in and visible. Non-negotiable.
  - Settings is a dialog opened from toolbar button, NOT a tab.
  - Settings button lives on far right of toolbar (⚙ Settings).
  - No emojis on buttons except the gear icon on Settings.
  - Sensitivity slider: no recommendations made — user-judgment based. Default configurable in Settings.
  - Version format: v0.x.0 through 1.0.0, then semantic (1.0.1, 1.1.0 etc.)
  - Remux supports MKV (mkvmerge) and MP4/M4V (ffmpeg). No mkvmerge for MP4.
  - Extract & Save filename: [video stem].[lang].[ext] e.g. Movie.eng.srt
    SDH tracks: Movie.eng.sdh.srt
  - File queue in Single File shows color only, no stale count badges.
  - SubForge is an accessibility tool — SDH removal and any feature that
    degrades the HoH experience must carry a visible warning and must never
    be enabled by default. Always default to the HoH-friendly option.
  - Track title normalization during remux is always on — no option.
    Encoder credits in track titles serve no user and actively hurt
    HoH users navigating subtitle menus.
  - Cleaning options are global (settings.json), not per-tab.
  - mkvmerge path lives in Settings > Paths, not in Video Scan tab.
